<objective>
DATA-04: deduplicate the corpus (exact + fuzzy near-dupes, D-11) BEFORE splitting, then produce
source-disjoint, class-aware 70/15/15 grouped splits (D-10) — the two operations that make the
held-out test set leak-free and realistic. Interface-first: these run on a DataFrame fixture so the
orchestrator (01-06) just composes them.

Purpose: A near-dup landing in both train and test = silent leakage; same outlet in two splits = model
learns the outlet not the content. Dedup-then-grouped-split closes both holes (RESEARCH Anti-Patterns).
This plan runs in Wave 3 (after 01-03) because the group-key derivation reads the confirmed per-source
label/column semantics produced by 01-03; sequencing them removes the same-wave race.
Output: `src/data/dedup.py`, `src/data/split.py`, and the dedup + split-disjoint stub tests turned green.
</objective>
