# AI-Driven Real-Time Detection of Fake News & Malicious Content

PMICS Project (Group 11), University of Dhaka. Classifies text as **real / fake / malicious**
using a hybrid of classical ML + a fine-tuned transformer, augmented by source-credibility,
writing-style, and external-verification modules combined via a decision-fusion layer, served
through a real-time Streamlit UI. Bilingual: **English + বাংলা**.

## Architecture
```
Input (text/URL) → preprocess (clean·normalize·lang-detect·tokenize)
   → Text classifier        (TF-IDF + LogReg/NB/RF ; XLM-RoBERTa primary ; BanglaBERT for bn)
   → Source credibility      (domain trust lists + heuristics)
   → Writing-style analysis  (clickbait/scam lexicon, ALL-CAPS, !!, links)
   → External verification   (NewsAPI trusted-source corroboration)
   → Decision fusion → label + confidence + explainability → UI
```
See `reports/architecture.png` (generated by `python -m src.report_assets`).

## Layout
| Path | What |
|------|------|
| `src/data/` | download + build 3-class bilingual corpus, preprocessing |
| `src/models/` | `classical.py` (TF-IDF+ML), `transformer.py` (XLM-R/BanglaBERT) |
| `src/modules/` | `credibility.py`, `style.py`, `verify.py` |
| `src/fusion/` | `fuse.py` — weighted fusion + confidence + explanation |
| `src/pipeline.py` | end-to-end text/URL → prediction |
| `app/streamlit_app.py` | real-time UI |
| `notebooks/` | `01` classical (CPU), `02` XLM-R fine-tune (Colab GPU) |
| `reports/` | metrics tables, confusion matrices, architecture diagram |
| `config.yaml` | datasets, model, fusion weights, API keys |

## Setup
```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt          # full set (incl. torch/transformers)
# CPU-only quick start (skip torch): pip install numpy pandas scikit-learn pyyaml \
#   pyarrow regex nltk trafilatura streamlit matplotlib seaborn requests beautifulsoup4
```

## Run
```bash
# 1. Build corpus (HF download; offline synthetic fallback if no network)
python -m src.data.build_corpus

# 2. Train classical baselines  → models/ + reports/classical_metrics.json
python -m src.models.classical

# 3. (Colab GPU) fine-tune transformer via notebooks/02 → download models/xlmr/

# 4. Predict from CLI
python -m src.pipeline --text "BREAKING: shocking miracle cure doctors hide!!!"
python -m src.pipeline --url  "https://www.bbc.com/news/..."

# 5. Report assets (tables, confusion plots, architecture diagram)
python -m src.report_assets

# 6. Launch UI
streamlit run app/streamlit_app.py
```

## Datasets (3-class merge)
- **real/fake** — English: `GonzaloA/fake_news`; Bangla: BanFakeNews
- **malicious** — `sms_spam` + `ealvaradob/phishing-dataset`

Downloads run at build time. If the hub is unreachable, an offline **synthetic** toy corpus is
generated so the full pipeline + UI still run for a demo.

## External verification
Set a free [NewsAPI](https://newsapi.org) key: `export NEWSAPI_KEY=...`. Without it, the module
degrades gracefully and its fusion weight is redistributed.
